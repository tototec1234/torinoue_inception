# Inception 学習メモ：NGINXのデーモン化と `daemon off;`

一次資料（本メモの根拠・深掘り用）:

- [nginx.org — Beginner’s Guide](https://nginx.org/en/docs/beginners_guide.html)（設定の全体像）
- [ngx_core_module — `daemon`](https://nginx.org/en/docs/ngx_core_module.html#daemon)（`daemon` ディレクティブの公式定義）

## 出発点となった疑問

DockerでNGINXを起動するとき、なぜ `exec nginx` だけではダメで
`daemon off;` が必要なのか？MariaDBは `exec mariadbd` だけで動くのに。

---

## 最初の認識（間違っていた部分）

> 「通常のプログラムはフォアグラウンドで動く。
>  バックグラウンドにしたければ `&` などのオプションが必要なはず。
>  だからMariaDBも特別ではなく、NGINXも同じはずでは？」

この認識は「**シェルから手動で起動するプログラム**」に対しては正しい。

しかしNGINXは違う。
**バックグラウンド化をシェルではなく、プログラム自身がやっている。**

---

## 前提知識：デーモンとは

- バックグラウンドで動き続けるプロセスのこと
- 端末（ターミナル）と切り離されて動く
- Webサーバー・DBサーバーなどのサーバープログラムに多い

---

## なぜNGINXはプログラム自身がデーモン化するのか（歴史的背景）

NGINXが生まれた2004年頃、Webサーバーは素のLinuxサーバー上で直接動かすものだった。

```
管理者がSSHでログイン
  └─ シェルから nginx コマンドを叩く
       └─ 管理者はSSHを切断したい
            └─ でもシェルを閉じると子プロセスも死ぬ（SIGHUP）
```

解決策：「起動したら自分でバックグラウンドへ逃げる」コードをNGINX自身に書いた。

---

## デーモン化の技術的な仕組み（Cレベル）

NGINXは起動時に内部でこれをやっている：

```c
pid_t pid = fork();   // 子プロセスを作る

if (pid > 0) {
    exit(0);          // 親プロセス（起動した本人）は即死
}

// 子プロセスが処理を引き継ぐ
setsid();             // 端末との関係を完全に断ち切る
// stdin/stdout/stderr を /dev/null へ
// → 以降は端末なしでバックグラウンドで動き続ける
```

ポイント：`fork()` で分裂し、**親が死んで子が仕事する**。

---

## なぜ `exec nginx` だけではDockerコンテナが死ぬのか

```
exec nginx を実行
  └─ シェルがNGINXプロセスに置き換わる（NGINXがPID 1になる）
       └─ NGINX内部で fork()
            └─ 子プロセスが仕事を引き継ぐ
       └─ 元のNGINX（PID 1）が exit(0) で死ぬ
            └─ DockerはPID 1が死んだのでコンテナを終了する 💀
```

Dockerの仕様：**PID 1が死んだらコンテナ全体が終了する。**

---

## `daemon off;` が何をするか

NGINXのソースコード内にある「デーモン化するかどうか」のフラグをオフにする。

```
daemon off; を設定すると
  └─ fork() をスキップ
       └─ NGINXがそのままPID 1に居座り続ける
            └─ Dockerコンテナが生き続ける ✅
```

---

## MariaDBとの対比

| | MariaDB (`mariadbd`) | NGINX（デフォルト） |
|---|---|---|
| 設計の前提 | フォアグラウンド動作も想定 | 端末から切り離す前提 |
| 起動時の `fork()` | しない | する（デーモン化） |
| Dockerでの扱い | `exec` だけでOK | `daemon off;` が必要 |

MariaDBも「特別なプログラム」ではなく、設計方針の違い。
どちらを選ぶかはプログラムの用途と歴史的な経緯による。

---

## まとめ

- 通常のプログラムはフォアグラウンドで動く → **正しい**
- NGINXはその例外：**自分でfork()してデーモン化するコードが内部にある**
- Dockerでは PID 1 が死 = コンテナ終了なので `daemon off;` で無効化する
- MariaDBはそもそもデーモン化しないので `exec` だけで動く
