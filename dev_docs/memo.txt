成果物の全体像
VM構成

VagrantfileはVirtualBox使用 → M2 MacではそのままでNGです（後述）
init.sh でVM内にDockerをインストールしている

コンテナ構成（全てAlpine 3.22ベース）
[NGINX]      ← openssl で自己署名証明書を生成、port 443
   ↓ fastcgi (port 9000)
[WordPress]  ← php-fpm + wp-cli でWordPress設定
   ↓ port 3306
[MariaDB]    ← entrypoint.shでDB・ユーザー作成
流れの概要

1.vagrant up → VMが立ち上がり、init.shでDockerインストール
2.VM内で make up → docker compose でビルド・起動
3.ブラウザで https://toruinoue.42.fr にアクセス

移植のオーバーヘッド
結論：30分〜2時間程度です。 ただし条件付きです。
変更が必要な箇所
Vagrantfileのprovider部分だけです：
ruby# M2 Mac（VMware Fusion）
config.vm.provider "vmware_fusion" do |vb|
  vb.memory = "4096"
  vb.cpus = 2
end

# 校舎（VirtualBox）に戻す場合
config.vm.provider "virtualbox" do |vb|
  vb.memory = "4096"
end
Docker関連のファイルは一切変更不要です。 Dockerfileもdocker-compose.ymlもそのまま動きます。
