## C 定義・運用・参照

### このファイルの役割

- **クイズ単独**の実施記録と、**EV/AC/見込み**の集計、**全フェーズのタスク表**を置く。`phase_plan.md` はスクロール負荷を下げるため、本ファイルへ切り出した。
- **以後、さらにファイルを分割するか**は**ドライバーに確認**すること。

### クイズ単独表の対象・列の意味

**タスク表に行が無い**（事前クイズ・事後クイズ・総合クイズのみを実施したセッション）を記録する。転記元は **`session_logs/NNNN_session_log_inception.md` の「実作業時間」**（0.5h 切り上げ後）。

- **実施 計画h**: セッションにかかる見込み時間。AI がクイズ MD を新規作成するときは問題数・難易度から **作成計画h** も見積もり、B3 表に記入する。
- **実施 実動h**: 実施後にセッションログから転記。未実施は **`-`**。
- **#0007 / #0012 / #0013**: ログに実動のみあるため、**実施 計画h＝実施 実動h** とみなして記入した。
- **未実施のフェーズ単位クイズ**: **セッション**列は **`-`**、**実施 計画h** は見込み、**実施 実動h** は **`-`**、**作成計画h** は MD 新規作成時の見込み（既存ファイルは **`-`**）。

**運用（B3）:** クイズ単独セッションが完了するたび、ドライバー確認のうえ **該当行を更新**する。**事前クイズとタスクが同一セッション**の場合は本表には載せず、タスク表の備考・セッションログ番号で扱う。

### 進捗管理（時間ベース）— 定義（MECE）

| 用語 | 意味 |
|------|------|
| **BAC** | 計画h合計。タスク表の「時間」列の合計 ＋「クイズ単独」表の **実施 計画h** 合計（クイズは単独表に載るセッションのみ。0100/0200 事前など**混在セッションのクイズ**は含めない）。 |
| **EV** | 完了済みh（計画）。**完了済み**に `[x]` がある **タスク**（`タスク X-Y` / `タスク 0-1`）の計画hの合計 ＋ **クイズ単独表**で **実施 実動h** が数値の行の **実施 計画h** の合計。 |
| **AC** | 完了済みh（実績）。完了タスクについて **実動** が数値なら実動、**`-`** なら計画「時間」を実績とみなす。完了クイズは **実施 実動h**。 |
| **見込み時間** | **AC + (BAC − EV)**。未完了分を計画どおり（100%）と仮定。**CPI / EAC は用いない**。 |
| **消化率** | **EV ÷ BAC**（％は **小数第 1 位**）。 |

**h の表示:** 見込み・残り・差分など **計算の最終結果**は **0.5h 単位で切り上げ**し、**小数点以下は表示しない**。数値の直後に単位 **`h`** を付ける。

**残り（未完了の計画ベース）** は **BAC − EV** を 0.5h 切り上げて表示。**見込み − BAC** は **AC − EV** と等しい。

**注:** フェーズ 1・2 事前クイズはクイズ単独表外のため集計に含めない。

### 運用（B1・B2）

本節は **セッション終了時**（完了チェック・実動転記のタイミング）に、`phase_plan.md` の「完了済み」と突き合わせて **再計算**する（ドライバー確認のうえ）。
