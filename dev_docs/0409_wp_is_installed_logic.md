## wp core is-installed の検証対象

このコマンドは、単に「ファイルがあるか」ではなく**設定されたDBに接続でき、かつテーブルが存在するか**をチェックしています。

### 接続の仕組み
* **参照ファイル:** `wp-config.php`
* **接続先:** `--dbhost` で指定されたサーバー（例: `mariadb`）
* **確認内容:**
 	1. データベースサーバーに疎通できるか？
 	2. 指定されたデータベース（例: `wordpress`）にログインできるか？
 	3. WordPressの基本テーブルが存在するか？

### 注意点
WordPressのソースコードを配置済みであっても、**DBサーバーが起動していない、あるいは認証情報が間違っている場合**、このコマンドは「インストールされていない」とみなしてエラーを返します。

具体例として、MariaDB サーバーが起動していない状態で `wp core is-installed` を叩くと、ターミナルには 「Error: Error establishing a database connection（データベース接続確立のエラー）」 というかなり目立つ警告が出ます。

これがあると、**スクリプトで「DBがまだ準備できていないなら待機する」といったループ処理**を書きたいときに、**画面がエラーメッセージで埋め尽くされ**る。

だからこそ、2>/dev/null でその「騒がしい警告」を黙らせる必要があるわけです。
