# WP-CLI (wp core) の正体について

### 1. 正体は「PHPスクリプト」
`wp core`（および `wp` コマンド全体）は、バイナリファイルではなく、**PHPで書かれたスクリプトファイル**です。

* **実体:** `wp-cli.phar` という名前の **Phar (PHP Archive)** 形式のファイルです。これは、複数のPHPファイルを1つにまとめた実行可能アーカイブです。
* **仕組み:** ファイルの先頭に `#!/usr/bin/env php`（シバン）が記述されているため、OSがPHPインタープリタを探して実行します。

### 2. PHP-FPM との関係
WP-CLIは **PHP-FPMを経由しません。**

* **PHP-FPM:** Webサーバー（Nginx等）からのリクエストを待機し、サイトを表示するための実行環境。
* **PHP-CLI:** ターミナルでコマンドを動かすための実行環境。

`wp` コマンドを実行したときは、PHP-FPMではなく **PHP-CLI上で動作し、WordPressの内部コードを直接呼び出すPHPプログラム**です。

## WP-CLIの動作構造

| 項目 | PHP-FPM (Web) | PHP-CLI (WP-CLI) |
| :--- | :--- | :--- |
| **起動のきっかけ** | ブラウザからのアクセス | ユーザーがコマンドを入力 |
| **実行環境** | 常にバックグラウンドで待機 | コマンド実行時のみ起動・終了 |
| **設定ファイル** | `php-fpm.ini` など | `php.ini` (CLI用) |

---

### 3. wp core は何をしているのか？

`wp core` は、WordPressのコアファイル群（`wp-load.php` など）を**直接読み込みます。**

* **ステップ1:** ユーザーが `wp core is-installed` と打つ。
* **ステップ2:** `wp` (PHPスクリプト) が起動する。
* **ステップ3:** `wp` が、実行ディレクトリにある `wp-config.php` を探し出し、データベース情報を読み取る。
* **ステップ4:** WordPressの内部関数を直接呼び出して、インストール済みかどうかを確認する。

> **ポイント:**
> Webサーバー（NginxやApache）を経由せず、OS上のPHPプロセスが直接データベースやファイルシステムとやり取りするため、Webサイトがダウンしていても操作が可能です。

## 補足：なぜ「バイナリ」っぽく見えるのか？

多くのチュートリアルで `chmod +x wp-cli.phar`（実行権限の付与）をして `/usr/local/bin/wp` に移動させるため、まるで `ls` や `cd` のようなネイティブなバイナリコマンドのように見えますが、実際には **「PHPで書かれた便利なツールを、コマンドっぽく使えるようにしているだけ」** なのです。

もし、サーバーに PHP-CLI が入っていなければ、WP-CLI は一切動きません。

WP-CLI を使っていて、Webサイト側（PHP-FPM）と設定が違ってハマった（例：メモリ制限やタイムアウトなど）経験はありますか？
